### What is AKKO?

AKKO (**A**nalytics **K**ernel, **K**eep **O**wnership) is a sovereign data and AI platform built on open-source components. It unifies the layers a regulated team needs — ingestion, storage, compute, AI, governance, observability — into a turnkey lakehouse that deploys via **Helm on Kubernetes** (local, sovereign cloud, or fully air-gapped).

| Layer | Purpose |
|-------|---------|
| **Ingestion & orchestration** | Schedule, retry, observe pipelines |
| **Storage & catalog** | Object store + open table format |
| **Compute & query** | Federated SQL + distributed jobs |
| **AI & ML** | Sovereign LLMs, vectors, experiment tracking |
| **Governance & security** | SSO, RBAC, audit, masking, lineage |
| **Observability** | Metrics, logs, traces, alerts |

### Why AKKO?

- **Data sovereignty** — every byte stays on your infrastructure (RGPD, DORA, NIS2, air-gapped capable).
- **No vendor lock-in** — open standards (Iceberg, S3, SQL, OAuth2). Every layer is replaceable.
- **AI-native** — local LLMs with an OpenAI-compatible gateway, MCP servers for sovereign AI agents (Claude, Cursor), AI SQL functions callable from any data source. Zero cloud-API dependency.
- **Fine-grained access** — row-level filtering and column masking propagated through SSO, consistent across every layer.
- **Kubernetes-first** — one Helm command on any conformant cluster.
- **Self-service catalog manager** — onboard any client source (PostgreSQL, MySQL, Oracle, Snowflake, Databricks, Hive, Kafka, MongoDB…) from the cockpit with zero downtime.
- **Full audit trail** — events from every layer aggregated into a single dashboard for compliance.
